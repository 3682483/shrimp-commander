贡献指南

CONTRIBUTING TO OPENCLAW

🤝 如何贡献

OpenClaw 是开源项目，欢迎所有人参与贡献！无论你是开发者、设计师、文档撰写者还是测试人员，都可以为项目做出贡献。

💡 贡献方式

  • 提交 Bug 报告或功能建议
  • 修复 Bug 或开发新功能
  • 创建和分享 Skill
  • 改进文档和教程
  • 翻译内容到其他语言
  • 帮助其他用户解决问题

📋 提交代码

1

Fork 仓库

在 GitHub 上 Fork openclaw/openclaw 到你的账户

2

创建分支

为你的修改创建一个新分支

git checkout -b feature/your-feature-name
3

编写代码

遵循代码规范，添加必要的测试和注释

4

提交 PR

Push 到你的 Fork，然后提交 Pull Request

git push origin feature/your-feature-name
5

代码审查

等待维护者审查，根据反馈修改代码

🧩 创建 Skill

Skill 开发流程

创建并发布你的 Skill 到 ClawHub：

openclaw skill create my-skill
# 编辑 SKILL.md 和相关文件
openclaw skill test
openclaw skill publish

详细教程请查看 Skill 开发指南

📝 代码规范

基本要求

  • 使用 TypeScript 编写代码
  • 遵循 ESLint 配置
  • 添加单元测试
  • 编写清晰的 Commit Message
  • 更新相关文档

Commit 规范

feat: 新功能
fix: Bug 修复
docs: 文档更新
style: 代码格式调整
refactor: 代码重构
test: 测试相关
chore: 构建/工具相关
